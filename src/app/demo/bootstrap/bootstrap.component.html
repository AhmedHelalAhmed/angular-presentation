What is Angular?

Angular is a development platform for building mobile and desktop applications. Angular lets you extend HTML's syntax to express your application's components clearly and succinctly. Angular's binding and Dependency Injection eliminate much of the code you would otherwise have to write.

Given an HTML file

Let's create an Angular app which replaces <hello-world> element with the application contents.

This can be done with 3 simple steps.

Creating angular app

  1. Create an angular component
  2. Create an angular module
  3. Bootstrap the module

Step 1: Create component

Angular component is just a class. Properties and behavior can be added inside.

Component Anatomy / Decorators

The class is adorned with a @Component decorator providing additional metadata to let the Angular know how to use it.

Decorators

Decorators are a new feature of TypeScript. (TODO:work in progress).

Decorators were inspired by a similar feature in Python language.

Component Anatomy / Selector

Selector defines the location of the component. When Angular renders this component, it'll find a <hello-world> HTML element in the document and will render the component inside of it.

Component Anatomy / Inline Template

Template defines the HTML code that the component generates.

If the amount of HTML grows out of hand, it's possible to use templateUrl instead and provide a path to a HTML file.

We need to put it in an NgModule

NgModules help us organize our code into cohesive blocks of functionality.

Module Anatomy

Like a component, a module is just a class.

Module Anatomy / The @NgModule Decorator

The NgModule decorator declares the AppModule class to be an angular module. The NgModule decorator accepts metadata describing the features of the module.

Module Anatomy / Browser Module

Because we're viewing the app in a browser, we pass BrowserModule into the imports array. The BrowserModule provides our AppModule with the capaility to render our components in the browser!

Module Anatomy / Declarations

The Declarations array specifies all components to be included in the module.

Components included in the same module live in the same space, and are aware of eachother.

Bootstrapping the Module

Bootstrapping the module start our application!

First, platformBrowserDynamic() tells Angular that we are operating in the browser. We then call bootstrapModule() with our AppModule so we can bootstrap it.

Bootstrapping the Module

Next, the bootstrapper will render the components found in the declarations array by inserting their templates into elements found in our index.html by matching the selectors defined in the @Component decorators. We have lift off!
